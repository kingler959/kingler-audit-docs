♾️ Endless Node Mechanics — How They Actually Work

Verified against program source: programs/sage/src/state/research.rs:165-179 (ResearchNode struct), character.rs:94-112 (add_node), character.rs:84 (combine_repeated on sync).

The hard constraint

Each research node has exactly ONE CharacterModifier payload. When you unlock level N, the engine applies that same modifier N times via combine_repeated(modifier, level). There is no per-level modifier table.

What this means: "Level 1 does X, level 2 does Y, level 3 does Z" on a single endless node is not possible without a program change. The original proposal had this wrong — corrected here.

What IS supported (zero program changes)

PatternHow it worksExampleUsed for
A. Scalar stacking Each level stacks the same CVC. Multiplicative fields grow exponentially; additive fields grow linearly. rare_mineral_discovery: {diamond: 1.05} → L10 = ×1.63 yield All 4 endless nodes in this proposal
B. Chained milestone nodes Create N sibling nodes (each max_level=1) in a chain. Each has its own unique modifier. T1 Gear → T2 Gear → … → T5 Gear (existing tier ladders) Optional Capstone Milestones (see below)

Optional: Pattern B Capstone Milestones

If you want per-level variety (not just bigger numbers), chain 3-5 sibling max_level=1 nodes off an endless track. Each capstone grants a unique CVC or tag:

Master Construction (endless, slots)
  └── Capstone I: Automated Refitting    → unlocks a new building category tag
      └── Capstone II: Advanced Logistics  → +1 crafting_job_concurrency_bonus
          └── Capstone III: Megastructure  → unlocks T6 hub placement (future)

These are regular one-time nodes, not endless. They give players discrete milestones alongside the endless grind. Not required — the two endless scalar tracks per career are sufficient for the 30-day target. Available if you want variety.

Why two endless nodes per career?

A single endless node with multiple CVCs forces players to pay for bonuses they don't want. Splitting by fantasy lets players specialize:

  • Mining: "rare specialist" (#82, high risk/reward) vs "volume industrialist" (#377, steady output)
  • Building: "bigger complex" (#137, more slots) vs "better complex" (#378, more output per slot)

Both use Pattern A scalar stacking → zero program changes required.

How They Interface — The Three-Pillar Supply Chain

The three careers form a supply chain, not parallel ladders. A solo player can do all three; an org can specialize and trade.

Identity Summary

CareerOwnsEndgame HookPlayer Fantasy
Mining Raw resource extraction from planets. Throughput maximization. Rare Minerals endless node — ever-better yields "I strip-mine star systems for profit."
Building Processing raw → components, hub infrastructure, terrain access. Master Construction endless — slots, speed, efficiency "I build the industrial backbone of the faction."
Crafting Deep assembly into ships/weapons/gear. Class + exotic + stimulant trees (already deep) "I forge the tools of war."
Key insight: Faction specialization lives in Building, not Mining. That's because faction-specific processors (Cryostabilin, Psyconine, Ballistene) are derived from faction-specific raw resources, and processors are Building-tier content. Mining extracts the raw; Building processes the faction-specific derivative. The faction identity is a Building mechanic.

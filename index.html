Starting Node #0

Starting Node grants every cargo category, five fleet slots, fleet size four, and max_ship_size: XxSmall. It grants neither productive nor residential plot capacity. Therefore the 34 starter Airbikes are usable, while every Claim Stake and Crafting Hab token remains unplaceable until capacity is unlocked.

Exact implication: the statement “gain at least six levels before placing a stake” is correct only for current generated programs data. Canonical SoT makes it twelve.

Crafting contract

Copper, not Copper Ore

The primary bundle gives 25,000 Copper #1033. It does not give Copper Ore #311. That is sufficient for the canonical starter craft:

Ammunition #0
input: 2 Copper
output: 1 Ammunition
value: 20
research_requirements: []
hab_required: false
usage_limit: 100

The Copper recipe itself consumes Copper Ore, but a fresh player does not need it because processed Copper is already supplied.

Recipe SoT gap. Current programs has 3,150 ungated/no-hab recipes. PR #807 proposes gates, while PR #180 records that recipe requirements, value, and duration still lack durable SoT ownership. A regen cannot guarantee the starter contract until that seam is closed.

Parallel public crafting remains the fastest gate route

PR #807 removes XP multiplication by output quantity, but public no-hab crafting has no per-player process cap. Recipe usage limits are global per recipe, not per Character.

75 separate Ammo processes → 80 seconds → 1,500 Crafting XP → proposed L12 → 2 renown

This consumes 150 Copper, 75 of the 1,000 faucet crew, and 0.00075 ATLAS in raw fees. The batch exploit becomes process fan-out.

Building dependency chain

other-career levels → renown → stake capacity → place stake → finalize hub → Building XP

Exactly 0 of 404 T1 building definitions are directly constructible from starter-bundle cargo. The bundle provides stakes, but ownership is not usability and stake placement is not an XP action.

Practical MUD starter hub

Terrestrial Central Hub Tier 1
25 Arco             ← mine at CSS asteroid
15 Copper Ore       ← mine at CSS asteroid
20 Framework        ← starter bundle
Building XP: 100 on finalize

Mining the 40 missing units has an ideal floor of ~153 seconds with one Airbike or ~31 seconds aggregate with five Airbikes, before loading, travel, resource splitting, and transactions. Seven of eight hub archetypes have all missing inputs on the MUD CSS asteroid; Volcanic additionally needs processed Neodymium.

Not a hard resource deadlock. The player can mine the missing materials. It is still not an immediate Building career because the action depends on another career, renown, stake placement, and hub finalization.

First-hours models

Current programs/main

  1. Onboarding claims starter cargo, 1,000 ATLAS, and 1,000 crew.
  2. Daily check-in supplies current #21’s Daily XP requirement.
  3. Start Ammunition quantity 25 with 25 crew.
  4. One second later: 500 Crafting XP → current Crafting L6 → one renown.
  5. Buy #21 directly from #0.
  6. Mine hub inputs, place stake, finalize hub → Building XP.

Canonical SoT + PRs #180/#807

  1. Start 75 separate one-crew Ammunition processes at the public starbase.
  2. After 80 seconds: 1,500 Crafting XP → proposed L12 → two renown.
  3. Buy #45, then #21.
  4. Mine hub inputs, place stake, finalize hub → Building XP.

A less transaction-heavy path uses 27 parallel Ammo processes for Crafting L6 plus five Airbikes mining to Mining L6 in an ideal ~8.6 minutes. Either way, Crafting parallelism—not the six-career experience—is the optimal gate route.

07

Recommended changes

P0 · Make the gate intentional

If every career is meant to start from Node #0, grant one T1 productive slot in Starting Node:

{
  "max_productive_plot_tier": "1",
  "max_productive_plots": 1
}

Change #21 from +20 plots to +19 if the intended post-unlock total should remain 20.

P0 · Guarantee one complete hub

Preferred: guarantee a Terrestrial tutorial plot and add 25 Arco + 15 Copper Ore. If plot archetype cannot be guaranteed, supply the small universal T1 hub kit listed in the Markdown source.

P0 · Close the recipe ownership seam

Author recipe requirements, XP value, duration/minimum, and hab requirement in durable SoT before regen. Add a structural assertion for Ammunition #0 rather than pinning drift-prone recipe counts.

P0 · Make Combat executable—or defer it honestly

Author nonzero typed damage on a starter XxSmall config and provide a legal training target. Otherwise, remove “all six careers immediately available” from onboarding claims.

P1 · Cap public Crafting progression

Add a per-Character public-process cap or a shared Crafting XP time budget. Do not use the recipe-global usage limit as a player pacing mechanism.

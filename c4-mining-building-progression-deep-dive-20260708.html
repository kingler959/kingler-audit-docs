1.2 The Renown Mechanic

renown_earned = floor(total_career_levels_gained_across_6_feeders / 6)

The 6 feeders are Pilot, Data Runner, Mining, Building, Crafting, Combat. Daily Check-In is NOT a feeder. Career levels accumulate forever; the divisor carries the remainder forward (so 7 levels = 1 renown + 1 carry).

Source: programs/sage/src/state/xp.rs:134 accrue_council_rank_from_career_levels(), DEFAULT_RENOWN_DIVISOR = 6.

3.2 What's MISSING from Building vs Mining

Asymmetry alert: Mining has 26 unlocks; Building has 13. Building is missing parallel structure for:
  • Tier-locked specialized buildings — Mining has explicit T1-T5 gear (#119-123); Building's T1-T5 structural complexity (#127-131) only gates processors/hubs/modules, not specialized faction buildings by tier.
  • Risk-zone ladder — Mining has 3 regional extraction tiers (#124-126: faction → enemy → deep space). Building only has Medium Risk (#135). No "High Risk Construction" or "Enemy Zone Construction" counterparts.
  • Hardness analog — Mining gates by resource hardness (#156-159). Building has no equivalent for "extreme surface" or "hostile atmosphere" construction (those are v2 placeholders).
  • "Rare discoveries" endless node — Mining has #82 (Unlimited). Building has no equivalent.

5. Level Cap — confirmed UNCAPPED, no program change needed

Verified 2026-07-08: There is no level cap anywhere — not in the program, not in the client.
  • Program logic (programs/sage/src/state/xp.rs:310-344): the expected_level function extrapolates past the last threshold using the last step size. level_threshold() (which WOULD error past the list length) is never called by game logic — only expected_level is, and it degrades gracefully.
  • Account sizing: XpDefinitions is unsized_type with a variable-length List<XpLevelThresholdInfo>. No fixed-size array, no u8 level field that would overflow.
  • Client (star-atlas-tech, searched): zero hardcoded 40 / maxLevel / LEVEL_MAX assumptions.
  • Update mechanism (c4-cli/src/init_game/xp.rs): UpdateXpThresholds with truncate: true replaces the entire threshold list in chunks of 10. Extending from 41 → 60 thresholds is a config change in level_thresholds.json + a re-seed — no program redeploy needed.
Conclusion: the "cap" is just how many thresholds are currently seeded (41 = L0 through L41). Adding L42-L60 is a data file edit.

8. Gaps, Risks, and Recommendations

8.1 Critical gaps

  1. Renown hard cap below CR tree cost. 40 max vs 46 needed. Must lower divisor or raise level cap.
  2. Building xp_value is uniform (100) across all 2020 buildings. No incentive to build T4/T5 structures vs T1 spam. Should scale: T1=100, T2=200, T3=400, T4=800, T5=1600 (or similar exponential).
  3. Mining xp_modifier is uniform (1) across all 3283 cargo types. Rare resources should reward more XP. Recommend: tier-scaled (T1=1, T2=2, T3=5, T4=12, T5=30) — rewards mining higher-tier veins.
  4. #117 Siphon Drill is a dead-end node. Per finalize/tables.mjs, classifyDrillBucket routes all fluid/liquid resources to #116 Fluid Extraction, so zero buildings carry the "Siphon Drill -" prefix. Fix: either add members or remove the node.

8.2 Asymmetries between Mining and Building

  • Mining has 26 unlocks; Building has 13. Building needs parallel depth.
  • Building lacks risk-zone ladder (only #135 Medium Risk; no High Risk or Enemy Zone equivalent).
  • Building lacks "hardness" analog (extreme surfaces, hostile atmospheres are v2 placeholders).
  • Building has no "endless node" equivalent of Mining's #82.

8.3 Fun factor assessment

Honest read on "is it fun?": The current shape has two fun-killers:

  1. The first 100 XP grind. A Class-1 starter earning 472 XP/hr needs 12 minutes of pure mining to hit L1 Mining. That's fine. But the next career (say Building) also needs 100 XP, and at 300 XP/hr that's 20 min. With 6 careers all needing L1 to bootstrap, the cold-start is 1-2 hours of varied grinding before the player has any career breadth. Daily Check-In helps but isn't a feeder.
  2. The L20+ wall. Past L20 the curve's exponential bite means each level takes longer than the last in absolute terms but the reward (a research node unlock) stays constant. Diminishing dopamine. Players will disengage around L25-30 when each level takes a week.

What would make it fun: front-load rewards (cheaper early levels), back-load prestige (expensive late levels that grant cosmetic/bragging rights rather than power), and make the "endless" nodes (#82 and a Building equivalent) the long-tail engagement loop with visible per-level effects.

8.4 Recommended action sequence (6:1 LOCKED, no program changes)

  1. Curve flatten (data only): regenerate level_thresholds.json with 1.06 exponential growth, extended from 41 → 60 thresholds. Use the SAGE Editor Suite → Council Rank XP tool.
  2. Tier-scaled XP rewards (data only): set non-uniform xp_modifier on cargo types (T1=1, T2=2, T3=5, T4=12, T5=30) and xp_value on buildings (T1=100, T2=200, T3=400, T4=800, T5=1600) + crafting habs.
  3. Activate #82 Rare Minerals as the first real "endless node" with a per-level rare_mineral_multiplier bump, and add a Building equivalent (#136 Master Construction Techniques).
  4. Re-seed via c4-cli: init_xp_definitions pushes the new threshold list on-chain in chunks of 10.
  5. Fix #117 Siphon Drill — populate members or mark v2.

Joseph's ruling: 6:1 renown divisor is immutable. All five items above are data/seed changes — no Rust program modification, no redeploy of the sage program. The 30-day target becomes achievable for active players; casual players will need ~45-60 days.
